# 裂缝检测数据集

## 简介

这是一个用于裂缝检测的目标检测数据集，主要用来识别两种不同方向的裂缝。数据经过增广处理，样本量充足。

## 数据规模

- 总图片数：**1846** 张
- 划分比例：8:1:1（训练集:验证集:测试集）

## 检测类别

本数据集包含2个类别：

| 类别 | 英文名 | 增广前数量 | 增广后数量 |
|------|--------|-----------|-----------|
| 纵向裂缝 | longitudinal_crack | 29994 | 46149 |
| 横向裂缝 | transversal_crack | 29362 | 50481 |

## 类别分布对比

### 增广前

![增广前类别统计](datasets/类别统计1.png)

原始数据中两个类别的样本数量比较接近，纵向裂缝约3万个，横向裂缝也约3万个，总共约6万个标注框。

### 增广后

![增广后类别统计](datasets/类别统计2.png)

经过数据增广后，纵向裂缝增加到约4.6万个，横向裂缝增加到约5万个，总共约9.7万个标注框，数据量更加充足。

## 数据处理说明

1. **图片尺寸标准化**：把所有图片统一缩放到640像素宽度，高度按原比例自动调整，标注框坐标同步换算。

2. **类别统计分析**：统计了各类别的样本数量，两个类别分布比较均衡。

3. **数据增广**：为了增加数据量，对两个类别都进行了数据增广，采用了随机缩放、随机旋转、亮度调整、对比度调整、饱和度调整、高斯模糊等方法。

4. **文件整理**：把所有图片和标签文件打乱顺序后重新编号

## 标注格式

YOLO格式，每个txt文件对应一张图片，每行表示一个目标框：
```
类别ID 中心点x 中心点y 宽度 高度
```
坐标值都是归一化后的（0到1之间）。

## 数据特点

- 每张图片平均包含约52个标注框，说明裂缝检测场景中目标比较密集
- 两类裂缝数量接近，模型训练时不会出现严重的类别偏向问题
